# CONSISTENCY_CHECKER — AI 电话客服系统

> 一致性检查规则
> 每次写入 project-center/ 文件前必须执行
> 最后更新：2026-06-29

---

## 一、检查范围

以下 5 个文件具有联动关系，修改其中一个时必须检查另外四个是否需要同步：

| 文件 | 内容 | 联动关系 |
|------|------|---------|
| `HANDOFF.md` | 近期决策表、阻塞项 | 决策变更 → 更新决策表 |
| `PRD.md` | 产品需求、功能状态 | 需求变更 → 更新功能状态 |
| `DECISION_LOG.md` | 所有决策记录 | 每次决策 → 追加记录 |
| `CHANGELOG.md` | 业务级变化 | 每次变更 → 追加记录 |
| `codebase_progress.md` | 任务状态 | 任务状态变化 → 更新 |

## 二、检查清单

每次修改任一个联动文件后，逐项检查：

- [ ] HANDOFF.md 决策表是否与 DECISION_LOG.md 一致？
- [ ] PRD.md 功能状态是否与 CHANGELOG.md 一致？
- [ ] DECISION_LOG.md 是否记录了本次决策？
- [ ] CHANGELOG.md 是否记录了本次变更？
- [ ] codebase_progress.md 是否需要更新？

## 三、冲突类型

| 类型 | 说明 | 处理 |
|------|------|------|
| CONFLICT | 同一信息在不同文件中表述矛盾 | 禁止写入，报告豪哥 |
| SEMANTIC_FORK | 同一决策在多个文件中有不同版本 | 禁止写入，报告豪哥 |
| STALE | 某文件未及时更新 | 允许写入，但需同步更新 |
| OK | 所有文件一致 | 允许写入 |

## 四、决策 ID 规范

格式：`D-YYYYMMDD-XXX`

- 同一决策禁止在多个文件中表达不同版本
- 新增决策必须使用新 ID
