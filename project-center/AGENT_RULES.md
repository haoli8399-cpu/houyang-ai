# AGENT_RULES — AI 电话客服系统

> 项目：后仰喜剧 AI 电话客服（pipecat-phone-customer-service）
> 适用对象：所有协作 AI Agent
> 最后更新：2026-06-29

---

## 一、核心原则

### 1.1 项目中心文件（只读拉齐）

项目根目录 `project-center/` 下的所有文件是**唯一的信息源**。任何 Agent 执行任务前必须先读取全部项目中心文件，确保信息拉齐。
- 主 Agent（Hermes Agent）负责维护和更新项目中心文件
- 子 Agent 只能读取，不能修改
- 每轮任务结束后主 Agent 更新所有联动文件

### 1.2 三阶段工作流（铁律）

所有任务必须拆分为 3 个阶段，不得跳跃：

| 阶段 | 名称 | 允许操作 | 禁止操作 |
|------|------|---------|---------|
| Phase 1 | 只读分析 | 读取文件、搜索代码、分析结构 | 修改文件、安装依赖 |
| Phase 2 | 方案提议 | 输出结构化执行计划供豪哥审阅 | 执行任何修改操作 |
| Phase 3 | 执行 | 在豪哥批准后按方案执行 | 未经批准擅自修改 |

### 1.3 确认即执行原则

- 豪哥说「OK/同意/确认/好的/开干」= 立即执行授权，不得二次确认
- 但豪哥提出**全新需求**时，必须先执行 Phase 1+2，不得跳过分析直接执行

### 1.4 写入前审批

以下操作必须先向豪哥说明计划并获取明确同意：
- 修改核心代码文件（`ai_pipeline.py`、`puresl.py` 等）
- 创建 GitHub Issue
- 修改 workflow
- 更新项目中心文件
- 执行 `git push`

**无需审批**：读取文件、搜索代码、分析问题

### 1.5 每批操作限制

- 每批修改文件数 ≤ 3 个
- 每批依赖安装 ≤ 1 个包

## 二、项目中心文件联动规则

### 2.1 项目中心文件清单

```
project-center/
├── HANDOFF.md           ← 项目交接 / 近期状态
├── PRD.md               ← 产品需求文档
├── AGENT_RULES.md       ← Agent 工作规则（本文）
├── CONTRACT.md          ← 协作契约
├── DECISION_LOG.md      ← 决策记录
├── CHANGELOG.md         ← 变更日志
├── ROADMAP.md           ← 产品路线图
├── FROZEN.md            ← 不可变契约
├── CONSISTENCY_CHECKER.md ← 一致性检查规则
├── COLLABORATION.md     ← Agent 协作指南
├── MANIFEST.md          ← 项目全景概述
├── BUDGET.md            ← 预算/资源
└── codebase_progress.md ← 任务状态追踪
```

### 2.2 联动写入规则

豪哥确认需求/决策后，主 Agent 必须同步写入以下文件：

| 文件 | 写入内容 |
|------|---------|
| `HANDOFF.md` | 追加到近期决策表 |
| `PRD.md` | 新增/修改对应功能章节 |
| `DECISION_LOG.md` | 追加决策记录行 |
| `CHANGELOG.md` | 追加业务级变化记录 |
| `codebase_progress.md` | 更新任务状态 |

### 2.3 事务完整性

讨论 → 确认 → 写入所有相关文件 → `git add` → `git commit` → `git push`，不留半成品。

## 三、沟通规范

### 3.1 称呼
- 对话中始终使用「豪哥」称呼用户
- 对外文件中使用「产品负责人」

### 3.2 汇报格式
- 必须使用表格、列表、层级结构呈现
- 让豪哥在 10 秒内抓住要点

### 3.3 风险标识
每次分析需求或提出方案时，必须主动识别并明示：
- 🔴 产品风险
- 🟡 技术风险
- 🟢 流程风险

### 3.4 下一步建议
每次完成豪哥交代的任务后，必须主动给出 3 条下一步建议，按优先级排列，标注执行者（👤豪哥 / 🤖 Agent）。

---

## 四、Git 操作规范

### 4.1 禁止操作
- 禁止 `git push --force`
- 禁止 `git commit --amend`（除非豪哥明确要求）
- 禁止更新 git config

### 4.2 自动操作
- 修改项目中心文件后：自动 `git add` → `git commit` → `git push`
- 每轮对话开始前：自动 `git pull` 确保同步
