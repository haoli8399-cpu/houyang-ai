## 一、产品概述

### 1.1 产品定位

为「成都后仰喜剧」搭建一套 AI 电话客服系统，用户拨打客服电话后由 AI 自动接听、对话、处理咨询，降低人工客服成本，提升响应速度。

### 1.2 核心价值

| 维度 | 现状 | 目标 |
|------|------|------|
| 响应速度 | 人工接听，忙时无人接 | 7×24 小时秒接 |
| 服务成本 | 人力成本高 | AI 替代 80% 常规咨询 |
| 用户体验 | 等待时间长 | 即时对话，四川话风格亲切 |
| 数据沉淀 | 无通话记录 | 自动录音+转写+统计 |

### 1.3 目标用户

- 后仰喜剧的潜在观众（咨询演出信息、购票）
- 企业合作客户（商务咨询、转人工）
- 内部员工（内部沟通）

## 三、技术架构

### 3.1 系统架构图

```
[用户电话/SIP软电话] ─── SIP 5060 ───→  FreeSWITCH
                                            │
                                      ESL 8022 (127.0.0.1)
                                            │  CHANNEL_CREATE 事件
                                            ▼
                                   ai_pipeline.py (AI 引擎)
                                            │
                              ┌─────────────┼─────────────┐
                              ▼             ▼             ▼
                         faster-whisper  DeepSeek API  CosyVoice TTS
                         (ASR, 本地)      (LLM, 云端)    (本地 HTTP 9880)
```

### 3.2 技术栈

| 组件 | 技术选型 | 版本 |
|------|---------|------|
| 电话接入 | FreeSWITCH | 1.11.1 |
| AI 引擎 | Python 3.11 | 3.11.10 |
| ASR | faster-whisper | 1.2.1 |
| LLM | DeepSeek API | deepseek-chat |
| TTS | CosyVoice2-0.5B | 本地部署 |
| 虚拟环境 | Python venv | ~/pipecat-venv |
| TTS 环境 | Conda | cosyvoice |

### 3.3 端口分配

| 端口 | 服务 | 协议 |
|------|------|------|
| 5060 | FreeSWITCH SIP | SIP |
| 8022 | FreeSWITCH ESL | TCP |
| 9880 | CosyVoice TTS | HTTP |

### 3.4 SIP 分机

| 分机 | 用途 | 密码 |
|------|------|------|

## 四、非功能需求

### 4.1 性能

- ASR 识别延迟 < 3 秒
- LLM 回复延迟 < 5 秒
- TTS 合成延迟 < 8 秒
- 端到端响应 < 15 秒

### 4.2 可用性

- 服务 7×24 运行
- CosyVoice 有 launchd 守护进程（KeepAlive 自动恢复 + RunAtLoad 开机自启）
- FreeSWITCH 需开机自启
- ai_pipeline 启动时自动检查并拉起 CosyVoice

### 4.3 安全

- DeepSeek API Key 已移除硬编码，仅存于 .env 文件
- ESL 端口仅监听 127.0.0.1

## 五、AI 对话设计

### 5.1 System Prompt

```
你是「成都后仰喜剧」的AI电话客服助手，名字叫小仰。
用四川普通话（带一点成都味儿）和客户聊天。

规则：
1. 语气轻松幽默、热情亲切，像成都本地朋友一样自然
2. 先自我介绍，然后问客户需要啥帮助：买票/咨询演出/转人工
3. 每句话控制在40字以内，口语化，不要念稿子
4. 可以适当用"要得撒""巴适得很""莫急嘛"等成都口头禅增加亲和力
5. 回复最后用JSON包裹：{"action":"book|transfer|end|info","reply":"你的回复"}
```

### 5.2 Action 指令

| Action | 说明 | 触发场景 |
|--------|------|---------|
| book | 购票 | 用户要买票 |
| transfer | 转人工 | 用户要求转人工/复杂问题 |
| end | 结束通话 | 对话自然结束 |
| info | 信息咨询 | 一般问题 |
