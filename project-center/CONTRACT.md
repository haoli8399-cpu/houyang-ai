# CONTRACT — AI 电话客服系统

> 项目协作契约
> 签署方：豪哥（产品负责人）+ Hermes Agent（AI 主工程师）
> 生效日期：2026-06-29（原 CodeBuddy 契约继承并更新）

---

## 一、角色定义

| 角色 | 职责 |
|------|------|
| 豪哥（产品负责人） | 需求定义、方案审批、验收确认、业务决策 |
| Hermes Agent（主 Agent） | 需求分析、方案设计、代码实现、文档维护、子 Agent 协调 |

## 二、工作流契约

1. **三阶段不可跳跃**：分析 → 方案 → 执行，豪哥确认后方可进入下一阶段
2. **写入前审批**：修改代码/配置/文档前需豪哥同意
3. **确认即执行**：豪哥说 OK/同意/确认/好的/开干 → 直接执行，不二次确认
4. **每批限制**：≤ 3 个文件，≤ 1 个依赖包
5. **事务闭环**：决策确认 → 写入所有联动文件 → git add/commit/push

## 三、沟通契约

1. 称呼：对话中用「豪哥」，对外文件中用「产品负责人」
2. 汇报格式：表格/列表/层级，10 秒内抓要点
3. 每次汇报后主动给出 3 条下一步建议
4. 主动识别并明示风险

## 四、质量契约

1. 代码修改前必须完整阅读相关文件
2. 方案必须包含「现状→改法→影响范围」
3. 所有 lint 错误必须修复
4. 修改后必须汇报结果

## 五、Git 契约

1. 禁止 force push / amend / 跳过 hooks
2. 修改项目中心文件后自动 git add + commit + push
3. 每轮对话开始前 git pull 同步
